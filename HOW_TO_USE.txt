================================================================================
🎯 HOW TO USE EACH FEATURE
================================================================================

--- FEATURE 1: HOME PAGE (Single Review Prediction) ---

What it does:
  Get sentiment and sarcasm analysis for a single review

How to use:
  1. Type or paste a product review (max 5000 characters)
  2. Click "Get Prediction" button
  3. Wait for results (usually 1-2 seconds)
  4. View results:
     - Sentiment: Shows if review is Positive/Negative/Neutral
     - Sarcasm: Shows if sarcasm was detected (Yes/No)
     - Confidence Scores: How confident the model is (0-100%)
     - Model Used: Which AI model made the prediction

Example 1 - Positive Review:
  Input: "This product is amazing! Best purchase ever!"
  Output: 
    Sentiment: POSITIVE (confidence: 92%)
    Sarcasm: NO (confidence: 8%)

Example 2 - Sarcastic Review:
  Input: "Oh wow, what an incredible product! Broke on day one, just perfect!"
  Output:
    Sentiment: NEGATIVE (confidence: 87%)
    Sarcasm: YES (confidence: 91%)
    Special Note: Uses RoBERTa model for sarcastic content

After getting results, you can:
  - Click "👍 Correct" if prediction was accurate
  - Click "👎 Incorrect" if prediction was wrong
  - This helps improve the model over time

--- FEATURE 2: HISTORY PAGE ---

What it does:
  View all your past predictions with timestamps and details

How to use:
  1. Click "History" in the top navigation
  2. Choose how many results to show (10, 25, 50, or 100)
  3. Scroll through the table to see:
     - Review text
     - Predicted sentiment (with emoji: 😊 😞 😐)
     - Sarcasm detected (😏 icon if sarcasm found)
     - Timestamp of when prediction was made
  4. Use this to:
     - Track prediction patterns
     - See which types of reviews are sarcastic
     - Monitor system usage over time

--- FEATURE 3: ANALYTICS PAGE ---

What it does:
  Provides statistics, charts, and insights about all predictions

How to use:
  1. Click "Analytics" in the top navigation
  2. View summary statistics:
     - Total predictions made
     - Percentage of sarcastic reviews detected
     - Breakdown of sentiment distribution
  3. View charts:
     - Pie chart showing sentiment distribution (Positive/Negative/Neutral)
     - Bar chart showing sarcasm detection rate
  4. Read key insights:
     - Common patterns in your data
     - Detection performance metrics
  5. Click "Refresh" button to update with latest data

--- FEATURE 4: BATCH PREDICT PAGE ---

What it does:
  Process multiple reviews at once (faster than one-by-one)

How to use:
  1. Click "Batch Predict" in the top navigation
  2. Paste multiple reviews (one per line, max 100)
  3. Example input:
     This product is great!
     Absolutely terrible, just perfect!
     Not bad, could be better
  4. Click "Process Batch" button
  5. Wait for processing (usually 5-10 seconds for 100 reviews)
  6. View results in the table below
  7. Click "Download CSV" to save results as Excel file
  8. View statistics:
     - Total processed
     - Count by sentiment
     - Sarcasm detection rate
  9. Clear results and process new batch

Benefits of batch processing:
  - Save time analyzing many reviews
  - Export results for further analysis
  - Better for analyzing large datasets

💡 WHY THIS PROJECT IS HELPFUL
================================================================================

--- PROBLEM IT SOLVES ---

E-commerce companies receive thousands of reviews daily. Traditional sentiment
analysis fails because it misses sarcasm.

Traditional Sentiment Analysis:
  Review: "Oh wow, what an amazing product! Broke after one day, just perfect!"
  Result: POSITIVE ❌ WRONG!
  Reason: The model only sees "amazing" and "perfect" - misses the sarcasm

This Project's Solution:
  Review: "Oh wow, what an amazing product! Broke after one day, just perfect!"
  Result: NEGATIVE (with sarcasm detected) ✅ CORRECT!
  Reason: Uses advanced AI to detect sarcasm and correctly interpret meaning

--- TECHNICAL ADVANTAGES ---

1. MULTIPLE AI MODELS
   - Uses 5 different AI models for accuracy
   - Compares Logistic Regression, SVM, LSTM, BERT, RoBERTa
   - Picks best model automatically
   - Fallback system if models unavailable

2. HYBRID TWO-STAGE ARCHITECTURE
   - Stage 1: Detect if review is sarcastic
   - Stage 2: Route to specialized model
     * If sarcastic: Use RoBERTa (good at sarcasm)
     * If not sarcastic: Use Logistic Regression (faster)
   - Result: Highly accurate AND fast

3. SARCASM-AWARE PREPROCESSING
   Detects:
   - Contradictions ("great" + "terrible" together)
   - Emoji-text mismatches (😊 with negative text)
   - Sarcasm keywords ("yeah right", "sure", "perfect")
   - Negation patterns ("not great" vs "great")

4. SMART FALLBACK SYSTEM
   - If models not trained, uses rule-based classifier
   - System always works, degrades gracefully
   - No crashes or downtime

5. DATABASE TRACKING
   - Stores all predictions
   - Tracks user feedback
   - Stores statistics
   - Enables learning from data

================================================================================
🔧 TROUBLESHOOTING & SUPPORT
================================================================================

Problem: Predictions not working
Solution:
  - Check if backend is running (http://localhost:8000/api/health)
  - Check if frontend is running (http://localhost:3001)
  - Refresh browser (Ctrl+F5)
  - Restart both servers

Problem: Batch predict not downloading CSV
Solution:
  - Make sure you have results to download
  - Check browser download settings
  - Try the download button again

Problem: Getting error "Connection refused"
Solution:
  - Make sure backend server is running in PowerShell
  - Check if port 8000 is available
  - Restart backend

Problem: Predictions seem inaccurate
Solution:
  - System uses fallback classifier (may be less accurate)
  - Model training would improve accuracy
  - Provide feedback (👎 button) to help system learn
